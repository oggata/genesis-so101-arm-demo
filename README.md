# SO-101 シミュレーター（Genesis版）

SO-101ロボットアームのシミュレーション環境です。  
キーボードによる手動操作と、PPO強化学習によるキューブ把持の学習・デモが行えます。

---

## セットアップ

### 1. ファイルを配置する

```
Genesis/
└── examples/
    └── so101_mac_viewer.py   ← ここに配置
```

### 2. 依存ライブラリをインストールする

```bash
pip install genesis-world stable-baselines3 gymnasium tensorboard opencv-python
```

---

## 実行方法

### 環境確認（キーボード手動操作）
引数なしで起動するとキーボード操作モードになります。  
学習前に環境が正しく動いているか確認するのに使います。

```bash
python so101_mac_viewer.py
```

### 学習してそのままデモ
```bash
python so101_mac_viewer.py --mode traindemo
```

### 学習のみ
```bash
python so101_mac_viewer.py --mode train
```

### 既存モデルでデモのみ
```bash
python so101_mac_viewer.py --mode demo
```

### デモのエピソード数を指定
```bash
python so101_mac_viewer.py --mode demo --episodes 10
```

### TensorBoardで学習曲線を確認
```bash
tensorboard --logdir genesis_so101_rl/logs
```

## キーボード操作マッピング

キーボード操作モードでは以下のキーで各関節を動かせます。

| キー | 動作 |
|------|------|
| `F1` / `F2` | Joint 0（ベース回転）+/- |
| `F3` / `F4` | Joint 1（肩ピッチ）+/- |
| `F5` / `F6` | Joint 2（肘）+/- |
| `F7` / `F8` | Joint 3（手首ピッチ）+/- |
| `F9` / `F10` | Joint 4（手首ロール）+/- |
| `-` / `=` | Joint 5（グリッパー）開/閉 |
| `U` | リセット（キューブをランダム再配置） |
| `ESC` | 終了 |

> **Macのファンクションキーについて**  
> `fn` キーを押しながらでないと反応しない場合は、  
> **システム設定 → キーボード → 「F1、F2などのキーを標準的なファンクションキーとして使用」** をオンにしてください。

### ゲームコントローラーを使う場合

コントローラーのボタンを上記キーにバインドする場合は、  
macOSのアプリ（[Joystick Doctor](https://www.generalarcade.com/joystickdoctor/) など）でキーマッピングを設定してください。

## カメラウィンドウ

キーボード操作モードでは、メインビューワーに加えて2つのカメラウィンドウがOpenCVで表示されます。

| ウィンドウ名 | 内容 |
|---|---|
| `OVERHEAD` | 真上からアーム全体を俯瞰する固定カメラ |
| `WRIST` | グリッパーの真上に追従するカメラ（グリッパー視点） |

ウィンドウの表示位置は `MAIN_WIN_W` の値（デフォルト: `1000`）を調整してください。

## 物体パラメータのカスタマイズ

`so101_mac_viewer.py` 内の以下の箇所でブロックの物性を変更できます。

```python
material=gs.materials.Rigid(
    rho=2000,              # 密度（kg/m³）
    friction=2.0,          # 床との摩擦係数
    coup_friction=2.0,     # グリッパーとの接触摩擦係数（把持力に直結）
    coup_restitution=0.3,  # 弾力（0.0〜1.0）
)
```

```python
morph=gs.morphs.Box(size=(0.025, 0.025, 0.025), ...)  # ブロックサイズ（メートル）
```

---

### 密度（rho）の参考値

密度はブロックの重さに直結します。サイズが 2.5cm 角の場合の質量目安も記載しています。

| rho（kg/m³） | 素材イメージ | 2.5cm角の質量目安 |
|---|---|---|
| 100 〜 200 | 発泡スチロール・軽木材 | 約 2〜8 g |
| 500 | 木材（松・杉） | 約 8 g |
| 700 | 木材（ oak・合板） | 約 11 g |
| 1000 | 水・ポリプロピレン | 約 16 g |
| 1200 | ABS樹脂・PLA | 約 19 g |
| 1400 | PVC・ナイロン | 約 22 g |
| 2000 | アルミニウム | 約 31 g |
| 2700 | アルミニウム（実測値） | 約 42 g |
| 7800 | 鉄・スチール | 約 122 g |
| 8900 | 銅 | 約 139 g |
| 11300 | 鉛 | 約 176 g |

---

### 摩擦係数（friction / coup_friction）の参考値

Genesisの摩擦係数は `0.01 〜 5.0` の範囲で設定できます。  
`coup_friction` はグリッパーがブロックを掴む際の接触摩擦で、把持のしやすさに最も影響します。

| 値 | 素材イメージ |
|---|---|
| 0.1 〜 0.3 | 氷・テフロン・潤滑油塗布面 |
| 0.3 〜 0.6 | 滑らかなプラスチック・ガラス |
| 0.6 〜 1.0 | 木材・乾燥した金属面（デフォルト相当） |
| 1.0 〜 2.0 | ゴム・シリコン・粗い金属 |
| 2.0 〜 4.0 | 高摩擦ゴム・砂面・ベルト素材 |
| 4.0 〜 5.0 | 極めて高摩擦（滑り止め素材など） |

---

### 弾力（coup_restitution）の参考値

`coup_restitution` はブロックが衝突・把持された際の跳ね返りを制御します。  
値が大きいほど弾みやすく、シミュレーションが不安定になりやすいため `0.3` 以下推奨です。

| 値 | 素材イメージ |
|---|---|
| 0.0 | 粘土・砂袋（完全非弾性） |
| 0.1 〜 0.2 | 木材・段ボール |
| 0.3 〜 0.4 | プラスチック・硬質ゴム |
| 0.5 〜 0.6 | テニスボール・軟球 |
| 0.7 〜 0.8 | スーパーボール・バスケットボール |
| 0.9 〜 1.0 | 完全弾性（理論値・実用では不安定） |

---

### アーム側の摩擦設定

ロボットアーム自体の摩擦も `add_entity` の `material` で設定できます。

```python
robot = scene.add_entity(
    material=gs.materials.Rigid(
        friction=2.0,       # アーム表面の摩擦
        coup_friction=2.0,  # ブロックとの接触摩擦（グリッパーの把持力に直結）
    ),
    morph=gs.morphs.URDF(file=URDF_PATH, pos=(0, 0, 0), fixed=True),
)
```

## 学習パラメータの調整

`train()` 関数内の以下の値を調整することで学習速度・品質を変更できます。

| パラメータ | デフォルト | 説明 |
|---|---|---|
| `N_ENVS` | `4` | 並列環境数。CPUコア数に合わせて増減 |
| `TOTAL_TIMESTEPS` | `60,000` | 総学習ステップ数。増やすほど学習が進む |
| `n_steps` | `2048` | 1回の更新に使うステップ数 |
| `batch_size` | `256` | ミニバッチサイズ |
| `learning_rate` | `3e-4` | 学習率 |
